## 声明一个变量的背后发生了什么

当你在一个`.Net`应用中声明一个变量，它会在内存中分配一些块空间。这块空间会包含三个东西：**变量名，数据类型和值**。

以上是一个简单的解释，但实际上，数据类型的不同将会导致其被分配到对应种类的内存空间。内存的分配类型有两种：**栈**和**堆**。在接下来的章节中，我们将会尝试深入理解这两种内存。

## 栈和堆

为了理解栈和堆，我们先理解以下代码的内部机制：

```csharp
public void Method1()
{
    // Line 1
    int i=4;

    // Line 2
    int y=2;

    //Line 3
    Class1 cls1 = new Class1();
}
```

让我们来解析这三行代码做了什么事情。

内存的分配与释放遵循**LIFO（Last In First Out）**的原则。换句话说，栈中内存最后分配的块会最先被释放。

- Line 1: 当这一行代码被执行，编译器会在栈中分配一小块内存空间。而栈负责管理你的应用所需要的运行内存。
- Line 2: 现在执行到了下一步。编译器会将这一块内存**分配在上一次内存分配的顶部**。可以理解为栈就是类似于将箱子垒起来的过程。
- Line 3: 在第三行，我们创建了一个对象。当这一行被执行，它在栈上创建了一个指针，然后实际的对象被保存在另一种不同的内存地址中，即是**堆**。堆不会管理应用所需的运行内存，它只是一个存储对象并可以在各处访问的地方。堆用来动态分配内存。

还有一个重要的点是，引用指针会被分配到栈上。语句`Class1 cls1;`不会给`Class1`的一个实例分配内存，而只是在堆上分配一个变量叫`cls1`(并将其设置为`null`)。当你使用`new`关键字时，才会在堆上分配内存。

在程序运行的最后，栈上的内存会被释放。换句话说，所有类似`int`数据类型的变量都会以`LIFO`的顺序被释放。

然而对于堆来说，程序的退出不会**立刻**导致堆内存的释放，而是在之后被**GC（Garbage Collector，即垃圾回收机制）**回收。

此时可能会有人问：为什么要区分出两种不同的内存分配方式呢？为什么不直接将所有的内存都分配到其中一种上呢？

你会发现，`.Net`的原始数据类型并不复杂且他们只被赋予了一个值，比如`int i = 0`。而对象的数据类型都很复杂，因为它们包含了对其他对象和原始数据类型的引用。换句话说，他们有对多个值的引用，而每个值都必须要被存储到内存。对象类型需要动态内存，而原始数据类型只需要静态内存。如果需要动态内存，它就会被分配到堆上，否则就会被分配到栈上。

## 值类型和引用类型

现在我们理解了栈和堆的概念，我们来看看值类型和引用类型。值类型在一个地方存储了数据和内存地址，而引用类型只存储了内存地址。

一个简单的整数类型变量`i`，它被赋值为另一个整数类型变量`y`，那么`i`和`y`都会被分配到栈上。

当我们将一个`int`类型变量赋值到另一个`int`类型变量上，它创建了一个完全不同的复制。换句话说，如果你改了其中一个变量的值，另一个并不会改变。
